## Arona คืออะไร?

**RAG (Retrieval-Augmented Generation)** คือเทคนิคที่ให้ AI สามารถค้นหาข้อมูลจากเอกสารแล้วนำมาตอบคำถาม เพื่อให้ได้คำตอบที่ถูกต้องและอ้างอิงแหล่งข้อมูลได้จริง

### วิธีทำงานแบบง่ายๆ:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  ผู้ใช้     │ ──▶ │  คำถาม     │ ──▶ │    Arona    │
│  User       │     │  Question   │     │    System    │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
                                                ▼
                                        ┌──────────────┐
                                        │ ค้นหาจาก    │
                                        │ เอกสารที่    │
                                        │ เกี่ยวข้อง   │
                                        └──────┬───────┘
                                               │
                                               ▼
                                        ┌──────────────┐
                                        │ นำข้อมูล    │
                                        │ ไปตอบคำถาม │
                                        └──────┬───────┘
                                               │
                                               ▼
                                        ┌──────────────┐
                                        │  ตอบกลับ    │
                                        │  + อ้างอิงที่มา│
                                        └──────────────┘
```

### การทำงานภายใน:

1. **ดึงเอกสาร** - Arona คัดลอกเอกสารจาก Elysia มา
2. **แบ่งส่วน** (Chunking) - แบ่งเนื้อหาเป็นส่วนๆ
3. **สร้าง Embedding** - แปลงข้อความเป็นเวคเตอร์ (ตัวเลข)
4. **เก็บในฐานข้อมูล** - เก็บใน ParadeDB (PostgreSQL + BM25 + Vector)
5. **อัปเดตอัตโนมัติ** - อัปเดตเฉพาะที่เปลี่ยน (Diff-aware)

## ทำไมต้องสร้างเอง?

### 1. บริการ RAG สำเร็จแพงเกินไป

เราหาบริการ RAG แบบ API ที่เหมาะกับงบประมาณและความต้องการแต่ไม่พบ

| บริการ | ราคาต่อเดือน | หมายเหตุ |
|---------|---------------|----------|
| Kapa | ~$1,000 | ราคาสูงมาก |
| Mendable | ~$500+ | ไม่ระบุราคาชัดเจน |
| Mintlify | ~$250 | แพงสำหรับ Startup |
| **Arona (DIY)** | **~$21** | **ประหยัดสุด!** |

สำหรับโปรเจ็กต์ใหม่ที่ต้องการคุมค่าใช้จ่าย ตัวเลือกแบบ DIY จึงเป็นทางเลือกที่ดีกว่า

### 2. เราลงทุนกับ VitePress แล้ว

เรามีเอกสารบน VitePress อยู่แล้ว พร้อมฟีเจอร์ต่างๆ เช่น:
- Interactive Tutorial
- Playground
- Custom Components

การย้ายไปบริการอื่นจะทำให้เสียข้อดีเหล่านี้

### 3. ความยืดหยุ่นในการปรับแต่ง

การสร้างเองทำให้:
- คุมข้อมูลได้ 100%
- ปรับแต่งได้ตามต้องการ
- ไม่ติด lock-in กับผู้ให้บริการ
- ลดค่าใช้จ่ายได้จริง

### เหตุผลที่ไม่ใช้ Re-ranking API

Re-ranking คือการ "จัดอันดับใหม่" ผลลัพธ์การค้นหาด้วย AI ซึ่ง:
- เพิ่มค่าใช้จ่าย (~$0.002 ต่อคำถาม)
- เพิ่ม Latency (~100-200ms)
- Hybrid Search (BM25 + Vector) ของเราเพียงพอแล้ว

## เทคโนโลยีที่ใช้

เลือกใช้ self-hosted เพื่อลดต้นทุน:

| Component | Technology | หน้าที่ |
|-----------|-------------|----------|
| **Backend** | Elysia (Bun) | API Server |
| **Database** | ParadeDB | BM25 + Vector Search |
| **Cache** | DragonflyDB | Redis-compatible Cache |
| **AI Main** | GPT OSS 120B | ตอบคำถามหลัก |
| **AI Small** | GPT OSS 20B | ปรับคำถามให้เป็นมาตรฐาน |
| **Embedding** | OpenAI Small | สร้างเวคเตอร์ (1536 มิติ) |
| **Monitoring** | Axiom | Logging & Tracing |
| **Security** | Turnstile | ป้องกันบอท |

## การติดตั้ง

### ขั้นตอนการติดตั้ง

```bash
# 1. ตั้งค่า Environment Variables
cp .env.example .env
# แก้ไขค่าต่างๆ ใน .env (DATABASE_URL, REDIS_URL, API keys)

# 2. เริ่ม Services ด้วย Docker
docker compose up -d

# 3. Setup Database และ Index เอกสาร
bun run scripts/setup.ts
# (แก้ไข script นี้ให้ตรงกับ repo เอกสารของคุณ)

# 4. เริ่ม Server
bun run dev
```

### Environment Variables ที่จำเป็น

```bash
# Database
DATABASE_URL=postgresql://user:pass@localhost:5432/arona

# Cache
REDIS_URL=redis://localhost:6379

# AI Providers
OPENAI_API_KEY=sk-xxx
OPENROUTER_API_KEY=sk-or-xxx

# Security
TURNSTILE_SECRET=0x...
CHALLENGE_SECRET=your-secret
AI_CHECKSUM_SECRET=your-checksum

# Optional: Monitoring
AXIOM_DATASET=arona
AXIOM_TOKEN=xaat-xxx
```
