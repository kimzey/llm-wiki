# 🛡️ เรียนรู้ Arona (ตอนที่ 9): ระบบป้องกันและส่วนเสริมต่างๆ (The Shield - Cache, Rate Limit, IP)

Arona ไม่ได้แค่ตอบคำถามเก่ง แต่ยัง "ดูแลตัวเองได้ดี" ด้วยครับ ในตอนนี้เราจะมาดูว่า Arona ป้องกันการใช้งานผิดวิธีและเพิ่มความไวได้อย่างไร

---

## 1. การป้องกันการใช้งานหนักเกินไป (Rate Limiting)
อยู่ใน `src/libs/rate-limit.ts`:
*   เรามีการจำกัดจำนวนคำถามต่อนาที (เช่น 10 คำถามต่อ 1 นาทีต่อ 1 User)
*   **เป้าหมาย:** ป้องกันการโดนยิง API จนค่าใช้จ่ายพุ่งสูง และป้องกันเครื่องค้าง (Denial of Service)

---

## 2. การตรวจสอบ IP Address
อยู่ใน `src/libs/ip.ts`:
*   ใช้ดึง IP ของ User เพื่อนำมาทำ Rate Limit และส่งให้ OpenAI (เป็นส่วนหนึ่งของการติดตามพฤติกรรมผู้ใช้เพื่อความปลอดภัย)

---

## 3. ระบบ Cache (การจดจำคำตอบเดิม)
Arona มีระบบ Cache 3 ระดับที่ทำงานร่วมกัน:
*   **`BurstCache` (`src/libs/burst-cache.ts`):** 
    *   จำคำตอบสั้นๆ (30 วินาที) ใน Memory 
    *   เอาไว้รับมือกับ User ที่กดปุ่มถามรัวๆ หรือหน้าเว็บที่เรียก API ซ้ำซ้อน
*   **`SemanticCache` (`src/modules/ai/libs/semantic-cache.ts`):** 
    *   จำคำถามที่มีความหมายคล้ายกัน (เช่น "How are you?" กับ "Are you okay?")
    *   ถ้าเคยตอบไปแล้ว ระบบจะดึงคำตอบเดิมมาทันทีโดยไม่ต้องเรียก AI (ประหยัดเวลาและประหยัดเงิน)
*   **`Redis` (`src/libs/redis.ts`):** 
    *   ตัวช่วยเก็บข้อมูลระยะยาวข้ามการรีสตาร์ทเซิร์ฟเวอร์

---

## 4. ระบบ Retry (การพยายามใหม่)
อยู่ใน `src/libs/retry.ts`:
*   ถ้าเรียก OpenAI แล้วล้มเหลว (เช่น เน็ตหลุด หรือ API ล่มชั่วคราว) ระบบจะพยายามเรียกใหม่เองโดยอัตโนมัติ 3 ครั้ง ก่อนจะยอมแพ้และบอก User ว่ามีปัญหา

---

## 5. การตรวจสอบความถูกต้อง (Validation)
อยู่ใน `src/modules/ai/model.ts` และ `src/modules/ingest/model.ts`:
*   ใช้ **Elysia `t` (TypeBox)** เพื่อตรวจสอบว่า Request ที่ส่งมาหน้าตาถูกต้องไหม (เช่น `source_type` ต้องเป็น markdown เท่านั้น) 
*   **จุดเด่น:** ระบบจะแจ้ง Error ทันทีถ้าข้อมูลผิดพลาด โดยไม่ต้องส่งไปประมวลผลให้เสียเวลา

---

**สรุป:** Arona คือระบบที่ถูกออกแบบมาให้ "ทนทาน" (Resilient) และ "ฉลาดประหยัด" (Efficient) ด้วยเครื่องมือเสริมต่างๆ เหล่านี้
