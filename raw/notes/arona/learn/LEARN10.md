# 🚀 เรียนรู้ Arona (ตอนที่ 10): การตั้งค่าและการใช้งานจริง (The Foundation - Docker & Env)

ในบทสุดท้ายนี้ เราจะมาดูสิ่งที่ต้องทำเพื่อให้ Arona เริ่มต้นทำงานและอยู่รอดในโลกความจริงได้ครับ

---

## 1. การตั้งค่าสภาพแวดล้อม (Environment Variables)
ในไฟล์ `.env` (เลียนแบบจาก `.env.example`) คือที่เก็บความลับและความชอบของระบบ:
*   **`OPENAI_API_KEY`**: หัวใจของ AI (ถ้าไม่มี AI จะไม่ทำงาน)
*   **`DATABASE_URL`**: พิกัดสำหรับคุยกับ ParadeDB
*   **`REDIS_URL`**: พิกัดสำหรับคุยกับ Dragonfly (Redis)
*   **`API_KEYS`**: รายชื่อ Key สำหรับผู้ใช้ที่ได้รับอนุญาต

---

## 2. โลกของคอนเทนเนอร์ (Docker)
อยู่ใน `docker-compose.yml`:
*   **`paradedb/paradedb`**: ฐานข้อมูลสุดล้ำที่รวมเอา SQL, Vector และ Full-text ไว้ด้วยกัน
*   **`dragonflydb/dragonfly`**: ตัวแทนของ Redis ที่เร็วกว่าและกินทรัพยากรน้อยกว่า (ใช้เก็บ Cache และ Session)

---

## 3. สคริปต์เตรียมความพร้อม (Setup)
ก่อนเริ่มใช้งานครั้งแรก เราต้องเตรียมพื้นที่ให้พร้อม:
*   **`bun scripts/setup.ts`**: สคริปต์นี้จะสร้างตาราง `sources` และ `chunks` พร้อมกับลง Extension `pgvector` และ `paradedb` ให้โดยอัตโนมัติ

---

## 4. โหมดการทำงาน (Development vs Production)
Arona มีความฉลาดในการปรับตัว (อยู่ใน `src/libs/flags.ts`):
*   **Dev Mode:** แสดง Swagger (OpenAPI) ให้เราดู API ได้ง่ายๆ และมี Log ละเอียด
*   **Production Mode:** ปิดหน้าจอแสดงผล API เพื่อความปลอดภัย และส่ง Log ไปที่ระบบ Monitoring (Axiom)

---

## 5. การดูแลรักษา (Maintenance)
*   **การอัปเดตเอกสาร:** เมื่อมีการเปลี่ยนเนื้อหาไฟล์ ให้ยิง Ingest อีกครั้ง (ระบบจะอัปเดตเฉพาะส่วนที่เปลี่ยนให้เองตามบทที่ 3)
*   **การเคลียร์ Cache:** ถ้า AI ตอบเพี้ยน เราสามารถสั่งล้าง Redis หรือ Semantic Cache ได้เพื่อให้ AI กลับมาคิดใหม่แบบสดใหม่

---

## 🏁 บทสรุปสุดท้าย

1.  **Ingest:** การนำเข้าและเตรียมข้อมูล

5.  **Shield:** ระบบป้องกันและการเพิ่มประสิทธิภาพ
6.  **Setup:** การติดตั้งและการตั้งค่า
