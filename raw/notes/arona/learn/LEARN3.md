## 1. ลายนิ้วมือดิจิทัล (Hashing)
ระบบใช้ **SHA-256** ในการสร้าง "ลายนิ้วมือ" ให้กับทุกส่วนของข้อมูล:
*   **Source Hash:** รหัสรวมของทั้งไฟล์ ถ้าไฟล์ไม่มีการเปลี่ยนแม้แต่ตัวอักษรเดียว รหัสนี้จะเหมือนเดิม
*   **Chunk Hash:** รหัสของแต่ละส่วนที่ถูกตัดออกมา (เช่น 1 ย่อหน้า) 

**หลักการ:** ถ้าเนื้อหาเปลี่ยนนิดเดียว รหัส Hash จะเปลี่ยนไปทันที!

---

## 2. ฟังก์ชัน `planIngestion()` ทำงานยังไง?
เมื่อเราตัดแบ่งไฟล์ใหม่เป็นชิ้นๆ (New Chunks) ระบบจะเอาไปเทียบกับชิ้นงานเดิมที่อยู่ใน Database (Existing Chunks) โดยใช้กฎดังนี้:

### กฎที่ 1: ถ้า Hash เหมือนเดิมเป๊ะ (Reuse)
*   **เหตุการณ์:** ชิ้นใหม่ที่ตัดมา มีเนื้อหาเหมือนกับชิ้นเดิมใน DB
*   **ผลลัพธ์:** ระบบจะบอกว่า **"ใช้ของเดิมได้เลย" (Reuse)**
*   **ข้อดี:** ไม่ต้องส่งไปให้ OpenAI เสียตังค์ทำ Embedding ใหม่ (ประหยัดค่า API ไปได้เยอะ!)

### กฎที่ 2: ถ้าลำดับเดิมแต่เนื้อหาเปลี่ยน (Update)
*   **เหตุการณ์:** อยู่ในลำดับเดียวกัน (Sequence เดียวกัน) แต่เนื้อหาในย่อหน้านั้นเปลี่ยนไป
*   **ผลลัพธ์:** ระบบจะบอกว่า **"ต้องอัปเดต" (Update)**
*   **ข้อดี:** แก้ไขเฉพาะจุดที่เปลี่ยน ไม่ต้องลบทิ้งทั้งหมด

### กฎที่ 3: ถ้าไม่มีในฐานข้อมูลเลย (Create)
*   **เหตุการณ์:** เป็นย่อหน้าใหม่ที่เพิ่มเข้ามา หรือไฟล์ใหม่แกะกล่อง
*   **ผลลัพธ์:** ระบบจะบอกว่า **"สร้างใหม่" (Create)**

### กฎที่ 4: ถ้าของเดิมใน DB หายไป (Delete)
*   **เหตุการณ์:** ย่อหน้าเดิมถูกลบออกไปจากไฟล์ต้นฉบับ
*   **ผลลัพธ์:** ระบบจะสั่ง **"ลบทิ้งจาก DB" (Delete)** เพื่อความสะอาดของข้อมูล

---

## 3. ทำไมต้องทำแบบนี้?
1.  **ประหยัดเงิน:** ค่า API ของ OpenAI คิดตามจำนวนคำ (Tokens) ถ้าเราไม่ส่งคำเดิมไปซ้ำ เราก็ประหยัดเงินได้
2.  **ความเร็ว:** การแก้ไขเฉพาะจุด (Update/Delete) เร็วกว่าการล้างตารางทิ้งแล้วลงใหม่ทั้งหมด
3.  **ความแม่นยำ:** ช่วยให้ข้อมูลใน Database ลิงก์กับเอกสารต้นฉบับได้อย่างถูกต้องเสมอ
