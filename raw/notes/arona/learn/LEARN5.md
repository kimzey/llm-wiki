# 🚀 เรียนรู้ Arona Ingest (ตอนที่ 5): แนวทางการพัฒนาต่อและตัวอย่าง (Next Steps & Implementation)

ยินดีด้วยครับ! คุณมาถึงขั้นตอนสุดท้ายแล้ว ในหน้านี้เราจะแนะนำวิธี "ลงมือทำ" และสิ่งที่ควรทำต่อไปครับ

---

## 1. วิธีรันโปรเจกต์และทดสอบ (Testing)
เพื่อให้แน่ใจว่าสิ่งที่เราทำไม่ไปพังส่วนอื่น:

*   **รัน Unit Test:** `bun test src/modules/ingest/service.test.ts`
    *   ใช้สำหรับทดสอบ Logic การวางแผน (Planning) และการเปรียบเทียบ Hash
*   **รัน API ใน Local:** `bun run src/index.ts`
    *   คุณสามารถใช้ Postman หรือ cURL เพื่อลองยิง Ingest ได้เลย

---

## 2. งานที่คุณอาจต้องทำ (Common Tasks)
ลองนึกภาพว่าคุณได้รับมอบหมายงานเหล่านี้:

### ก. การเพิ่มประเภทไฟล์ใหม่ (เช่น PDF)
1.  สร้างไฟล์ใหม่ใน `src/libs/extractors/pdf.ts`
2.  เรียกใช้ไลบรารีอย่าง `unpdf` เพื่ออ่านเนื้อหาจาก PDF
3.  กลับไปที่ `src/modules/ingest/service.ts` เพื่อเพิ่มเคส `source_type === 'pdf'`

### ข. การเปลี่ยนวิธีการตัดแบ่ง (Chunking Strategy)
1.  ไปที่ `src/libs/chunker.ts`
2.  แก้ไขฟังก์ชัน `chunkByHeading()` ถ้าอยากให้มันตัดตามความยาวแทนหัวข้อ
3.  ปรับแต่ง `maxChunkSize` เพื่อให้เหมาะสมกับประเภทข้อมูล

---

## 3. สคริปต์ช่วยงาน (Utility Scripts)
ในโฟลเดอร์ `scripts/` มีของดีซ่อนอยู่:
*   `scripts/setup.ts`: สำหรับเตรียมฐานข้อมูลและตาราง
*   `scripts/list-data.ts`: ดูว่าใน DB มีข้อมูลอะไรอยู่บ้าง

---

## 4. ข้อควรระวัง (Pitfalls)
*   **API Key:** อย่าลืมตั้งค่า `.env` ให้มี `OPENAI_API_KEY` ไม่งั้นขั้นตอน Embedding จะล้มเหลว
*   **Database:** ตรวจสอบให้แน่ใจว่า Docker (PostgreSQL) ทำงานอยู่
*   **Permissions:** ถ้าคุณค้นหาข้อมูลแล้วไม่เจอ ลองเช็คดูว่า `department` ใน Request ตรงกับข้อมูลที่ Ingest ไว้หรือไม่

---

## 🎓 บทสรุปสุดท้าย
