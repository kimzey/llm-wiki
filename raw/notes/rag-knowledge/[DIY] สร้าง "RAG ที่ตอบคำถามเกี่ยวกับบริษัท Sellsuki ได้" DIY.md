สร้าง "RAG ที่ตอบคำถามเกี่ยวกับบริษัท Sellsuki ได้"

โดยให้มันอ่านเอกสารบริษัททั้งหมด แล้วตอบจากเอกสารจริง ไม่เดา

  

usecase

ปัจจุบัน:

พนักงานมีคำถาม → ถาม HR / ถามเพื่อน / เปิดไฟล์หาเอง

ช้า, ได้คำตอบไม่ตรง, HR เสียเวลาตอบคำถามซ้ำๆ

  

หลังทำเสร็จ:

พนักงานมีคำถาม → ถาม AI ใน LINE/Slack/Web → ได้คำตอบทันที 24/7

เร็ว, ตรงประเด็น, อ้างอิงแหล่งที่มาได้, HR ไม่ต้องตอบซ้ำ

  

ก่อนทำ:

❌ พนักงาน: "ลาพักร้อนกี่วัน?" → ถาม HR → HR ตอบเอง → ช้า

❌ พนักงาน: "WiFi password อะไร?" → ถามรอบข้าง → ไม่มีใครจำได้

❌ พนักงาน: "สร้าง order ใน Sellsuki ยังไง?" → เปิดหา docs เอง → หาไม่เจอ

❌ Dev: เขียนโค้ด → ไม่รู้ business rule → ถามคนอื่น → รอ

  

หลังทำ:

✅ พนักงาน: "ลาพักร้อนกี่วัน?" → ถาม LINE Bot → "10 วัน/ปี" ทันที

✅ พนักงาน: "WiFi password อะไร?" → ถาม Slack Bot → ได้ทันที

✅ พนักงาน: "สร้าง order ยังไง?" → ถาม Web Chat → ได้ขั้นตอนครบ

✅ Dev: เขียนโค้ด → ถามใน Cursor → ได้ business rule + ตัวอย่างโค้ด

✅ HR: ไม่ต้องตอบคำถามซ้ำๆ → มีเวลาทำงานอื่น

  

Tech

จะมีการทดลองการทำ chunks หลายๆ แบบ , พร้อมแนบ Metadata ด้วย สำหรับการ filter,serach,จำกัดสิทธิ์

การทำ Database & Index (ของ Rag ต้องทำยังไง)

จะยังไม่ใช้ Framework พวก LangChin จะลองทำด้วยตัวเองก่อนให้เข้าใจ

มีการทำ Caching เช่น Semantic Cache

รองรับการ update ข้อมูล ถ้ามีการ update ข้อมูล ต้องไปใช้ข้อมูลล่าสุด เช่น อาจจะ Chunk ใหม่เข้าไป แต่ถ้า Embedding แล้ว มันคือข้อมูลเดิม ก็ไม่ต้อง Embedding ซ้ำข้อมูล Chunk นั้นอีกรอบ
