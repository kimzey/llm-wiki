## พื้นฐาน: context.Context ใน Go

`context.Context` เป็น interface มาตรฐานของ Go สำหรับส่งข้อมูลข้ามฟังก์ชันโดยไม่ต้องแก้ signature เช่น:

- deadline / timeout
- cancellation signal
- key-value values (trace span, request ID, user info ฯลฯ)

```
HTTP Request
    │
    ▼
middleware (inject span → context)
    │
    ▼
handler ← รับ context ที่มี span อยู่ข้างใน
    │
    ▼
useCase → repository → database
         (ส่ง context ต่อกันตลอด chain)
```

ทุกชั้นส่ง context ต่อกันเพื่อให้ OTel สร้าง trace tree ที่ถูกต้อง

---

## Fiber มี context กี่ตัว?

Fiber (ซึ่งใช้ fasthttp ข้างใต้) มี context **สองแบบ** ที่ต่างกันสิ้นเชิง:

### 1. `c.Context()` → `*fasthttp.RequestCtx`

]]```go
// fiber/ctx.go
func (c *Ctx) Context() *fasthttp.RequestCtx {
    return c.fasthttp
}
```

- คือ HTTP request context ของ fasthttp ระดับ low-level
- implement `context.Context` ได้ แต่ **ไม่มี** ค่า middleware inject ไว้
- ใช้สำหรับดึงข้อมูล HTTP (headers, body, IP) ไม่ใช่สำหรับ tracing
- **ไม่มี OTel span ใดๆ อยู่ใน context นี้**

### 2. `c.UserContext()` → `context.Context`

```go
// fiber/ctx.go
func (c *Ctx) UserContext() context.Context {
    ctx, ok := c.fasthttp.UserValue(userContextKey).(context.Context)
    if !ok {
        return context.Background()
    }
    return ctx
}

func (c *Ctx) SetUserContext(ctx context.Context) {
    c.fasthttp.SetUserValue(userContextKey, ctx)
}
```

- คือ Go standard `context.Context`
- **middleware inject ค่าเข้ามาได้** ผ่าน `c.SetUserContext(ctx)`
- นี่คือช่องทางมาตรฐานที่ library ต่างๆ ใช้ส่ง OTel span

## `otelfiber.Middleware()` ทำงานอย่างไร

ดู source จริงของ `github.com/gofiber/contrib/otelfiber/v2@v2.2.3/fiber.go`:

```go
return func(c *fiber.Ctx) error {
    // 1. เริ่มจาก UserContext ปัจจุบัน
    savedCtx, cancel := context.WithCancel(c.UserContext())

    // 2. Extract trace context จาก HTTP headers (W3C Trace Context / B3)
    //    เช่น traceparent: 00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01
    ctx := cfg.Propagators.Extract(savedCtx, propagation.HeaderCarrier(reqHeader))

    // 3. สร้าง span ใหม่ผูกกับ context
    ctx, span := tracer.Start(ctx, spanName, opts...)
    defer span.End()

    // 4. *** KEY LINE *** inject context ที่มี span เข้า UserContext
    c.SetUserContext(ctx)

    // 5. เรียก handler ถัดไป — handler จะได้ context ที่มี span แล้ว
    if err := c.Next(); err != nil { ... }

    // 6. cleanup หลัง handler เสร็จ
    defer func() {
        c.SetUserContext(savedCtx)
        cancel()
    }()
}
```

**บรรทัดสำคัญคือ `c.SetUserContext(ctx)`** (line 131)

middleware inject span เข้า `UserContext` ก่อนเรียก handler ดังนั้น handler ที่เรียก `c.UserContext()` จะได้ context ที่มี active span อยู่ข้างใน

## `getSpanContext(c)` ผิดอย่างไร

```go
// span.go (ที่ลบไปแล้ว)
func getSpanContext(c *fiber.Ctx) context.Context {
    ctx, ok := c.Locals("fiber-otel-tracer").(context.Context)
    if !ok {
        return c.Context()  // fallback
    }
    return ctx
}
```

มีปัญหา **2 ชั้น**:

### ปัญหาที่ 1: อ่านผิด key

`otelfiber` เก็บค่าใน Locals ด้วย key นี้:

```go
// otelfiber/fiber.go line 21-22
const tracerKey = "gofiber-contrib-tracer-fiber"

// line 98
c.Locals(tracerKey, tracer)  // เก็บ Tracer object ไม่ใช่ context
```

แต่ `getSpanContext` อ่านด้วย key `"fiber-otel-tracer"` ซึ่ง **ไม่มีอยู่**

### ปัญหาที่ 2: type ผิด

ถึงแม้ key จะตรง สิ่งที่ `otelfiber` เก็บใน Locals คือ `trace.Tracer` (object สำหรับสร้าง span)
ไม่ใช่ `context.Context` (context ที่มี span อยู่ข้างใน)

```go
// type assertion นี้ fail เสมอ เพราะ type ไม่ตรง
ctx, ok := c.Locals("fiber-otel-tracer").(context.Context)
// ok = false เสมอ
```

### ผลลัพธ์: fallback ไป `c.Context()` เสมอ

```go
if !ok {
    return c.Context()  // *fasthttp.RequestCtx — ไม่มี OTel span
}
```

ทุก handler ได้รับแค่ fasthttp context ธรรมดา **span ไม่เคย propagate ถึง use case / repository เลย**

## ภาพรวมเปรียบเทียบ

```
HTTP Request มาถึง
│
├─ otelfiber.Middleware() ทำงาน
│   ├─ Extract trace headers
│   ├─ สร้าง span
│   └─ c.SetUserContext(ctx)  ← inject span เข้า UserContext
│
├─ Handler ถูกเรียก
│   │
│   ├─ [เก่า] getSpanContext(c)
│   │   ├─ c.Locals("fiber-otel-tracer") → nil (key ผิด)
│   │   ├─ type assertion fail
│   │   └─ return c.Context()  ← fasthttp context ไม่มี span
│   │                              useCase ได้ context ว่างเปล่า
│   │                              ❌ ไม่มี trace ใดส่งไป Jaeger/Tempo
│   │
│   └─ [ใหม่] c.UserContext()
│       └─ return context ที่ otelfiber inject span ไว้แล้ว
│          useCase ได้ context ที่มี span
│          ✅ trace propagate ถูกต้อง Jaeger/Tempo ได้รับ span
│
└─ Response ส่งกลับ
```
